gRPC Load Balancing

展示 gRPC 長連線特性如何導致 L4 LB 流量失衡，以及各種現代解法的運作原理。

傳統 L4 Load Balancer (TCP層)

L4 LB 只看 IP 和 Port，以「TCP 連線 (Connection)」為單位進行分配。

因為 gRPC 基於 HTTP/2，會**重用同一條長連線 (Keep-Alive)**。一旦連線建立，客戶端所有的 gRPC Request 都會沿著這條「唯一的水管」湧向同一台後端伺服器 (Server A)。

結果：Server A 被打爆，Server B 和 C 卻閒置，完全失去負載平衡的作用！
